# Sen development sessions

A personal archive of Sen development sessions captured as patch series. Each session is a coherent unit of work
(a topic, a branch, a few days of effort) stored as one or more patches that can be re-applied to the upstream Sen
tree later.

This repo was originally a different project (an ncurses experiment) and is being repurposed; expect renames and
restructuring as the archive settles.

## Layout

Each session lives under `sessions/<YYYY-MM-DD>-<topic>/` and bundles:

- one or more `*.patch` files generated by `git format-patch` (or a single mbox with multiple commits);
- a `README.md` with the session's intent, the source branch, the **current status against `origin/main` /
  `origin/feat/term-component`**, and the `git am` invocation to re-apply.

The status line is the part to read first: it tells you whether the patches still represent live work, have
been merged exactly, or have been superseded by a different design upstream.

## Applying a session

From the root of an `airbus/sen` checkout:

```sh
git am /path/to/this/repo/sessions/<session-dir>/*.patch
```

Sessions that touch overlapping files have an order suggested by their date prefix; apply older sessions first
unless the per-session README says otherwise.

## What's not in this archive

Patches whose work has been integrated into `origin/main` or `origin/feat/term-component` byte-for-byte are not
kept here — for example, the `fix[core/test/base]: restore failed-check handler in test teardown` patch landed
exactly as `b433597` on `feat/term-component` and is omitted. The `feat[term]: integrate sen::lang::query for SQL
editor UX` patch from the 2026-04-24 SQL session is also omitted, since the term component was redesigned in
`99c3a07` along a different axis and that patch's approach is no longer the right shape.
